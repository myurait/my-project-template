# Supporting Features

Use this directory for planning-ready feature documents that sit below product epics and represent near-term roadmap candidates.

## Rules

- A supporting feature must point to at least one parent epic.
- Create a supporting feature only when the item is likely to be compared for the next roadmap window.
- A supporting feature must have concrete blockers, roadmap-order context, validation scenarios, and failure signals.
- Keep distant deferred requests in `01-feature-backlog.md`.
- Do not use this directory as a generic storage area for every future request.
